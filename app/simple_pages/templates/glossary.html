Introduction
What is Class & Object?

Class: Class is a "blueprint" for creating objects.
Ex. To create a class, use the keyword class
class Calculation:

Object: Objects is run time instance of class Ex. cal is a object of class Calculation
cal = Calculation()

Constructor: Constructors are generally used for instantiating an object. The task of constructors is to initialize(assign values) to the data members of the class when an object of the class is created. In Python the __init__() method is called the constructor and is always called when an object is created.
Ex.
class Calculation:
   def __init__(self):

Static Property: Static means, that the member is on a class level rather on the instance level. Variables declared inside the class definition, but not inside a method are class or static variables.
Ex.
class Calculations:
   """Calculations class manages the history of calculations"""
   history = []

Static Method: A class method is a method that is bound to the class and not the object of the class. @staticmethod decorator to create a static method in python.
Ex.
class Calculations:
   """Calculations class manages the history of calculations"""
   history = []
   @staticmethod
   def clear_history():
      """clear the history of calculations"""
      Calculations.history.clear()
      return True

Instance Property: An instance attribute is a Python variable belonging to one, and only one, object. This variable is only accessible in the scope of this object and it is defined inside the constructor function, __init__(self,..) of the class.
Ex.
class ExampleClass:
   def __init__(self, instance_attr):
     self.instance_attr = instance_attr

Instance Method: An instance method can access and even modify the value of attributes of an instance. It has one default parameter
Ex.
class ExampleClass:
   def __init__(self, instance_attr):
     self.instance_attr = instance_attr
   # Instance Method
   def instanceMethodTest(self, instance_attr):
     return self.instance_attr

Unit Test: A test case is the individual unit of testing. It checks for a specific response to a particular set of inputs. unittest provides a base class, TestCase, which may be used to create new test cases.
Ex.
def test_calculation_add_get_result_method():
   """Testing the Calculator add """
   tuple_list = (1, 2)
   calculation = Addition.create(tuple_list)
   assert calculation.get_result() == 3

Test fixtures: A test fixture represents the preparation needed to perform one or more tests, and any associated cleanup actions.
Ex.
@pytest.fixture
def setup_addition_calculation_fixture():
   """define a function that will run each time you pass it to a test"""
   values = (1, 2)
   addition = Addition(values)
   History.add_calculation(addition)
